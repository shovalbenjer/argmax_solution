# Definitive Project Plan: SOTA-Validated, Visually-Driven Dietary Classifier

## Project Vision
To engineer a production-grade, auditable, and highly traceable dietary classification system. This system will embody state-of-the-art principles by featuring a deterministic normalization engine for quantitative analysis, Ollama for efficient local LLM inference, MLflow for comprehensive experiment tracking, and a dedicated Jupyter notebook (`task.ipynb`) for interactive, visually-driven analysis of the results.

## Core Architecture: The Hybrid Retrieval Cascade
This architecture balances speed, accuracy, and auditability.

1.  **Ingredient Parsing**: `ingredient-parser-nlp` extracts `quantity`, `unit`, and `name` from raw ingredient strings.
2.  **Deterministic Normalization Engine**: The `ingredient_processor` uses the parsed components to query the `knowledge_graph.db`. It performs deterministic calculations to find the true nutritional values (e.g., total carbs in "2 cups of flour"). This is the quantitative core of the system.
3.  **LLM Judgment (Teacher/Student)**:
    *   **Teacher (via Ollama)**: Creates the nuanced ground truth dataset for evaluation.
    *   **Student (Qwen via Ollama)**: Classifies new recipes based on the rich, structured context generated by the engine.
4.  **Traceability (MLflow)**: Tracks all experiments, ensuring reproducibility and a clear audit trail.
